# OGC Maps

Modern, open-source tools for geospatial web mapping built on **OGC API standards**. We develop component libraries, mapping applications, and infrastructure for building scalable, standards-based mapping systems.

---

## Featured Projects

### [storybook-components](https://github.com/ogc-maps/storybook-components)
A production-ready React component library and configuration-driven map ecosystem. Build feature-rich mapping applications with pre-built UI components, data hooks for OGC APIs, and comprehensive validation.

**Includes:**
- React component library with Storybook documentation
- OGC API data hooks (Features, Tiles, Coverages)
- Config-driven architecture with Zod validation
- Full-featured demo map with admin configuration interface
- Docker infrastructure with PostGIS and OGC API endpoints

**Stack:** React · TypeScript · TailwindCSS v4 · MapLibre GL JS · Storybook 8 · Vite

---

### [gunnison-county-map](https://github.com/ogc-maps/gunnison-county-map)
Interactive web map for Gunnison County GIS data. Showcases real-world OGC API Features integration with a modern React frontend.

---

### [stac-higher](https://github.com/ogc-maps/stac-higher)
A modern interface for browsing, searching, and managing SpatioTemporal Asset Catalogs (STAC). Full CRUD operations, interactive visualization, and extensible API support.

**Stack:** Astro 6 · React 19 · MapLibre · TanStack Query · shadcn/ui

---

## Why OGC API Standards?

We believe in open, interoperable geospatial data. OGC API standards enable seamless integration across tools and platforms, reducing vendor lock-in and building a richer ecosystem for geospatial applications.
